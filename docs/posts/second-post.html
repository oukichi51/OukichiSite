プロジェクト概要

プロジェクト名

OukichiSite

目的

このプロジェクトは、自己紹介や技術ブログを公開するための個人ウェブサイトを構築することを目的としています。

ファイル構成

プロジェクトは以下のような構成になっています：

  • /docs/posts/: 各投稿のHTMLファイルを格納
  • /components/: ヘッダーやフッターなどの共通コンポーネント
  • /dist/: CSSやJavaScriptのビルド済みファイル
  • /scripts/: クライアントサイドのスクリプト

主な機能

このサイトは以下の機能を提供します：

  • 自己紹介ページ
  • 技術ブログの投稿と管理
  • レスポンシブデザインによるモバイル対応

今後の展望

今後は、コメント機能やダークモードの実装を検討しています。

GitHubPagesでTailwindcssを使用したブログを構築してみた

本記事では、GitHub Pagesを利用してTailwind CSSを活用したブログを構築する方法について解説します。 静的サイトジェネレーターを使わず、HTMLとCSSを直接編集するシンプルなアプローチを採用しました。

使用した技術

  • GitHub Pages: 静的サイトのホスティング
  • Tailwind CSS: ユーティリティファーストのCSSフレームワーク
  • HTML: コンテンツのマークアップ
  • JavaScript: コンポーネントの動的読み込み

構築手順

  1. GitHubリポジトリを作成し、GitHub Pagesを有効化
  2. Tailwind CSSをインストールし、設定ファイルを作成
  3. HTMLファイルを作成し、Tailwind CSSを適用
  4. 共通コンポーネント（ヘッダー、フッター）をJavaScriptで動的に読み込む仕組みを実装
  5. 完成したサイトをGitHub Pagesにデプロイ

工夫した点

Tailwind CSSのユーティリティクラスを活用して、レスポンシブデザインを簡単に実現しました。 また、共通コンポーネントを動的に読み込むことで、コードの重複を減らし、メンテナンス性を向上させました。

テスト環境

このプロジェクトでは、Dockerを使用してローカル環境でのテストを行っています。Dockerを利用することで、環境構築を簡略化し、開発者間での環境の一貫性を保つことができます。
